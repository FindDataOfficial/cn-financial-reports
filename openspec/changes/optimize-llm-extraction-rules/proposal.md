## Why

当前基于规则 + LLM 的指标抽取在部分“非三大表字段/附注类指标”上存在明显缺口：规则无法定位到正确章节，导致 LLM 输入上下文缺失或噪声过高，从而抽取失败或取值不稳定。随着公司年报/半年报/季报的版式差异与章节命名差异增大，单一规则很难覆盖全部场景，需要一套可持续迭代的“缺口发现 → 位置定位 → 规则修正 → 回归验证”闭环。

## What Changes

- 新增“规则缺口审计”流程：从既有规则与输出结果（out/）中识别无法通过规则访问或经常抽取失败的指标，并生成待修复清单与定位线索。
- 优化 report 型指标规则的章节选择器：面向年报/半年报/季报分别补齐更稳健的章节/别名集合，并提供 fallback 策略以适配不同公司披露结构。
- 强化“位置/章节定位”能力：在报告中定位指标可能出现的章节（如“主要会计数据和财务指标”“管理层讨论与分析”“财务报表附注”等）并输出可复用的定位结果，作为 LLM 抽取输入的上下文来源。
- 增加回归验证：以 out/ 目录中的样例结果为基线，验证修正后的规则能稳定命中正确章节并提升抽取成功率；若 out/ 样例不足，补充下载少量报告作为测试样本。

## Capabilities

### New Capabilities
- `rule-gap-audit`: 对规则可达性与抽取成功率做系统化审计，产出“缺失/失败指标清单 + 建议章节/别名 + 修复优先级”，用于驱动规则迭代。
- `report-section-map`: 建立面向年报/半年报/季报的章节标题与常见别名映射，用于 selector 统一、fallback 与跨公司版式适配。

### Modified Capabilities
- `indicator-rules`: 扩展并规范 report 型（LLM）规则的 `source.selectors` 定位策略，使其对多种报告类型与章节命名更鲁棒。
- `indicator-position-extract`: 强化定位结果的可解释性与可复用性（定位到章节/页/标题片段），并为 LLM 抽取提供更干净的上下文输入。
- `llm-indicator-extract`: 调整抽取前的上下文准备逻辑，使其优先使用定位到的章节内容，并在失败时按策略降级。

## Impact

- 影响配置与规则：indicator_rules.json 中 `source_type:"report"` / `extractor:"llm"` 相关规则的章节 selector 与覆盖面会变化。
- 影响抽取流程：LLM 抽取前的“章节获取/裁剪/拼接”逻辑与失败降级策略会调整。
- 影响测试与样例：会新增/更新基于 out/ 的回归验证数据与测试夹具，确保规则改动可量化评估。
